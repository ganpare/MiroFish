## ⚡ 概要

**MiroFish** は、マルチエージェント技術を利用した次世代AI予測エンジンです。現実世界から提供されたシード情報（最新ニュース、政策草案、金融シグナルなど）を元に、高精度の並行デジタル世界を自動構築します。その空間内で、独立した感情・長期記憶・行動ロジックを持つ何千ものスマートエージェントが自由にインタラクションし、社会的進化を遂げます。あなたは「神の視点」から動的に変数を注入し、未来の軌跡を精密に推論することができます—— **デジタル環境で未来の予行演習を行い、何百万回ものシミュレーションの果てに最適な決断を下しましょう**。

> あなたがすべきこと：シード資料（データ分析レポートや面白い小説のストーリーなど）をアップロードし、自然言語で予測の要件を記述するだけです</br>
> MiroFishが返すもの：詳細な予測レポートと、深くインタラクションできる高精度のデジタル世界

### 理念

MiroFish は、現実をマッピングする「群知能の鏡」の構築に取り組んでいます。個体間のインタラクションから生じる集団の創発を捉えることで、従来の予測の枠組みを突破します：

- **マクロレベル**：意思決定者のための予行演習としての役割を果たし、リスクゼロでの政策・PRの試行を可能にします
- **ミクロレベル**：一般ユーザーのためのクリエイティブなサンドボックスを提供します——小説の結末の推論から、想像力の果てを探るまで、すべてを楽しく簡単に体験できます

真面目な予測から遊び心のあるシミュレーションまで、あらゆる「もしも」の結末を可視化し、万物の予測を現実にします。

## 🔄 ワークフロー

1. **グラフ構築**: 現実のシード抽出 & 個体および集団記憶の注入 & GraphRAGの構築
2. **環境構築**: エンティティ関係の抽出 & ペルソナ生成 & 環境設定用エージェントへのシミュレーションパラメータ注入
3. **シミュレーション**: デュアルプラットフォーム並列シミュレーション & 予測要件の自動解析 & 時系列記憶の動的更新
4. **レポート生成**: 豊富なツールセットを持つReportAgentが、シミュレーション後の環境と深くインタラクション
5. **詳細インタラクション**: シミュレーション世界内の任意の個体や、ReportAgentとの対話

## 🚀 クイックスタート

### オプション1：ソースコードによるデプロイ（推奨）

#### 前提条件

| ツール | バージョン | 説明 | インストールの確認 |
|------|---------|-------------|-------------------|
| **Node.js** | 18以上 | フロントエンド実行環境。npmを含む | `node -v` |
| **Python** | 3.11以上、3.12以下 | バックエンド実行環境 | `python --version` |
| **uv** | 最新版 | Pythonパッケージマネージャー | `uv --version` |

#### 1. 環境変数の設定

```bash
# 設定ファイルのテンプレートをコピー
cp .env.example .env

# .envファイルを編集し、必要なAPIキーを入力
```

**必須となる環境変数：**

```env
# LLM API設定（OpenAI SDK形式をサポートするすべてのLLM APIを利用可能です）
# 推奨: Alibaba Qwen-plus モデル（Bailian プラットフォーム経由）: https://bailian.console.aliyun.com/
# 消費量が大きいため、最初はシミュレーション回数を40回未満に設定してお試しください
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud 設定
# 通常の利用であれば、無料の月間枠で十分です: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. 依存関係のインストール

```bash
# 全ての依存関係を一括インストール（ルート + フロントエンド + バックエンド）
npm run setup:all
```

また、ステップごとにインストールすることも可能です：

```bash
# Node の依存関係をインストール（ルート + フロントエンド）
npm run setup

# Python の依存関係をインストール（バックエンド、仮想環境を自動生成）
npm run setup:backend
```

#### 3. サービスの起動

```bash
# フロントエンドとバックエンドの両方を起動（プロジェクトルートで実行）
npm run dev
```

**サービスURL：**
- フロントエンド: `http://localhost:60613`
- バックエンド API: `http://localhost:43037`

**個別に起動する場合：**

```bash
npm run backend   # バックエンドのみ起動
npm run frontend  # フロントエンドのみ起動
```

### オプション2：Dockerデプロイ

```bash
# 1. 環境変数の設定（ソースコードでのデプロイと同様）
cp .env.example .env

# 2. イメージをプルしてコンテナを起動
docker compose up -d
```

デフォルトでルートディレクトリの `.env` を読み込み、ポート `60613`（フロントエンド）と `43037`（バックエンド）をマップします。

> 高速化のためのミラーアドレスは `docker-compose.yml` にコメントとして記載されています。必要に応じて置き換えてください。
